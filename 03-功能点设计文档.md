# Windows桌面自动化平台 —— 功能点设计文档

本文档对每个功能模块的输入输出、参数、交互细节和异常处理做具体设计，作为开发阶段的直接依据。所有"Block"均按架构文档中的Schema+Handler规范实现。

---

## 模块一：动作类 Block（操作屏幕）

### 1.1 鼠标点击 `click`

| 项 | 内容 |
|---|---|
| 参数 | `x`(int), `y`(int), `button`(left/right/middle), `click_type`(single/double), `move_duration`(ms，移动过程耗时，0为瞬移) |
| 输出 | 无（纯动作类节点） |
| 实现 | `pyautogui.click(x, y, button=button)` |
| 边界情况 | 坐标超出屏幕范围时应提前校验并报错，而非交给系统处理产生未定义行为 |
| DPI处理 | 坐标统一使用"逻辑像素"，内部按当前DPI缩放比例换算为物理像素后再调用API |

### 1.2 鼠标拖拽 `drag`

| 项 | 内容 |
|---|---|
| 参数 | `coordinate_mode`(screen_abs / window_client / virtual_norm)、`from_x`/`from_y`/`to_x`/`to_y`、`duration`(ms)；窗口模式可带 `window_target` 或起终点各自的 `from_window_target` / `to_window_target` |
| 实现 | 经 `resolve_point` 解析后 `pyautogui.dragTo`；`window_client` 随目标窗口移动回放 |

### 1.3 键盘输入 `key_press` / `type_text`

| 项 | 内容 |
|---|---|
| 参数(key_press) | `keys`(数组，支持组合键如 `["ctrl","c"]`) |
| 参数(type_text) | `text`(string，支持变量插值)，`interval`(每字符间隔ms) |
| 实现 | `pyautogui.hotkey(*keys)` / `pyautogui.typewrite(text, interval)` |

### 1.4 延时等待 `delay`

| 项 | 内容 |
|---|---|
| 参数 | `ms`(int) |
| 说明 | 最基础的流程控制节点，MVP阶段必须有 |

## 模块二：识别类 Block

### 2.1 区域颜色识别 `color_detect`

| 项 | 内容 |
|---|---|
| 参数 | `x`, `y`（单点取色）或 `region`[x1,y1,x2,y2]（区域取平均色/主色） |
| 输出 | `color`(hex字符串，如 `#FF0000`) |
| 实现 | mss截图指定区域 → Pillow读取像素/计算区域主色 |
| UI交互 | 提供"取色器"小工具：用户点击"选取"按钮后，鼠标变为十字光标，点击屏幕任意位置，自动回填坐标和颜色预览 |

### 2.2 颜色匹配条件 `if_color_match`

| 项 | 内容 |
|---|---|
| 参数 | `region`/`point`, `target_color`(hex), `tolerance`(0-255，颜色欧氏距离容差) |
| 输出 | `matched`(bool) |
| 说明 | 这是"识别+判断"合二为一的条件节点，内部先调用取色逻辑，再和目标色比对，避免用户还要额外接一个if节点做简单比较（虽然底层也支持通用if节点+变量比较的组合方式，两种方式并存） |

### 2.3 OCR文字识别 `ocr_recognize`

| 项 | 内容 |
|---|---|
| 参数 | `region`[x1,y1,x2,y2], `lang`(中文/英文/自动) |
| 输出 | `text`(string), `confidence`(float), `boxes`(每个文字块的坐标，供进阶用途) |
| 实现 | RapidOCR，模型本地加载，做单例缓存避免每次识别都重新加载模型 |
| 性能优化 | 识别区域尽量小，减少无关区域拖慢识别速度；提供"识别间隔"限制，避免高频循环中每次都全量识别 |
| 边界情况 | 区域内无文字时返回空字符串而非报错；置信度过低时（可配置阈值）视为未识别到 |

### 2.4 文字匹配条件 `if_text_contains`

| 项 | 内容 |
|---|---|
| 参数 | `region`, `expect_text`(支持"包含"/"完全匹配"/"正则"三种匹配模式) |
| 输出 | `matched`(bool), `actual_text`(string) |

### 2.5 图像模板匹配 `find_image`

| 项 | 内容 |
|---|---|
| 参数 | `template_image`(用户框选保存的小图路径), `search_region`(可选，缩小搜索范围), `threshold`(匹配相似度阈值0-1) |
| 输出 | `found`(bool), `x`, `y`(匹配中心坐标) |
| 实现 | OpenCV `cv2.matchTemplate` |
| UI交互 | 提供"框选保存模板图"工具，用户框选屏幕区域后自动截图保存为模板文件，关联到该节点 |

## 模块三：流程控制类 Block

### 3.1 条件分支 `if_condition`

| 项 | 内容 |
|---|---|
| 参数 | `expression`（变量比较表达式，如 `{{node_003.text}} == "登录成功"`，或直接引用某个bool型节点输出） |
| 分支 | `then`(node_id), `else`(node_id，可选) |
| 说明 | 底层表达式求值需限定安全的比较语法（不做任意代码eval，避免安全风险），支持 `==`, `!=`, `>`, `<`, `contains` 等有限操作符 |

### 3.2 多分支 `switch`

| 项 | 内容 |
|---|---|
| 参数 | `variable`(要判断的变量引用), `cases`(数组，每项含 `value` 和 对应 `node_id`), `default`(node_id) |

### 3.3 固定次数循环 `loop_n`

| 项 | 内容 |
|---|---|
| 参数 | `times`(int) |
| 结构 | `body`(循环体入口node_id), `next`(循环结束后跳转) |
| 内部状态 | 执行引擎需在context中维护当前循环计数器，key建议为 `__loop_{node_id}__counter`，避免嵌套循环变量冲突 |

### 3.4 条件循环 `loop_while`

| 项 | 内容 |
|---|---|
| 参数 | `expression`(同if条件表达式) |
| 结构 | `body`, `next` |
| 边界情况 | 需提供"最大循环次数"保护参数（如默认10000次强制退出），防止死循环导致程序失控 |

### 3.5 无限循环（常驻监控）`loop_forever`

| 项 | 内容 |
|---|---|
| 参数 | `exit_condition`(可选表达式，满足则退出), `check_interval_ms`(每轮间隔) |
| 说明 | 用于"持续监控屏幕状态"类场景；界面上需要有明显的"停止"按钮，且该类节点在画布上应有醒目视觉标识（如红色边框），提醒用户这是长驻运行节点 |

### 3.6 子流程调用 `call_subflow`（第二期）

| 项 | 内容 |
|---|---|
| 参数 | `subflow_path`(已保存流程的路径；Inspector 内从流程库下拉选择，无需系统文件对话框)、`inherit_variables`、`input_map`、`output_map` |
| 说明 | Handler 同步嵌套执行另一份 `.flow.json`；列表来自数据目录流程库，当前打开流程自身不会出现在选项中 |

## 模块三点五：系统类 Block（出桌面 IO）

失败时返回 `ok=false` + `error`，**不抛异常中断整条流程**，可用条件节点判断后继续。

### 3.7 HTTP 请求 `http_request`

| 项 | 内容 |
|---|---|
| 参数 | `method`(GET/POST/PUT/PATCH/DELETE), `url`, `headers`(keymap), `body`, `timeout_sec`, `encoding` |
| 输出 | `ok`, `status`, `body`, `error`, `headers` |
| 说明 | 使用标准库 `urllib`；HTTP 2xx 视为 `ok=true`，非 2xx 仍带回响应体 |

### 3.8 剪贴板 `clipboard`

| 项 | 内容 |
|---|---|
| 参数 | `action`(read/write), `text`(写入时) |
| 输出 | `ok`, `text`, `error` |
| 说明 | 通过 tkinter 访问系统剪贴板（与应用内复制 API 共用实现） |

### 3.9 文件读写 `file_io`

| 项 | 内容 |
|---|---|
| 参数 | `action`(read/write/append), `path`（可手输或「浏览」调系统对话框；读=打开，写/追加=另存为）, `content`, `encoding`(默认 utf-8) |
| 输出 | `ok`, `content`, `path`, `error` |
| 说明 | 读取有大小上限（2MB），超限返回明确 error；写入会自动创建父目录 |

### 3.10 执行命令 `run_command`

| 项 | 内容 |
|---|---|
| 参数 | `command`, `args`(空格分隔或 JSON 数组), `cwd`, `timeout_sec`, `shell`(默认 false) |
| 输出 | `ok`, `exit_code`, `stdout`, `stderr`, `error` |
| 说明 | 默认非 shell；超时或非零退出码 → `ok=false`，仍带回已捕获输出 |

### 3.11 系统通知 `notify`

| 项 | 内容 |
|---|---|
| 参数 | `title`, `message`, `play_sound` |
| 输出 | `ok`, `error` |
| 说明 | Windows 下通过 PowerShell 弹出 Toast；失败时用系统提示音兜底 |

### 3.12 Python 脚本 `python_script`（F-25）

| 项 | 内容 |
|---|---|
| 参数 | `code`(脚本正文，`ui: python_code` Monaco 编辑器), `inputs`(keymap，注入到脚本 `inputs` 字典) |
| 输出 | `ok`, `result`, `error`, `printed` |
| 注入 | `context`(只读)、`inputs`、`out`(可写 dict；写入 `out["result"]` 等) |
| 安全 | 受限 `exec`；`import` 白名单：`json/math/re/datetime/time/random/collections/itertools` |
| 说明 | 异常或非法 import → `ok=false`，不中断整条流程；编辑器提供示例插入与关键词补全（Ctrl+Space） |

示例：

```python
out["result"] = inputs.get("x", 0) + 1
print("value", out["result"])
```

### 3.13 用户自定义积木目录（F-25）

| 项 | 内容 |
|---|---|
| 路径 | `{数据目录}/user_blocks/*.py` |
| 规范 | 每个文件导出 `SCHEMA` + `handler`（与内置积木相同）；跳过 `_` 前缀 |
| 冲突 | `type` 与内置重名则跳过，不覆盖 |
| 分类 | Schema 可写 `category`；缺省为「自定义」 |
| UI | 设置 →「自定义积木」：内置 Monaco 编辑器（示例/补全）、新建/保存、打开目录、刷新积木 |
| 示例 | 目录为空时自动写入 `example_echo.py` |

### 3.14 等待窗口 `window_wait`

| 项 | 内容 |
|---|---|
| 参数 | `title` / `process_name` / `class_name`（由选取写入）、`timeout_sec`、`poll_ms` |
| 输出 | `ok`, `found`, `title`, `pid`, `process_name`, `error` |
| UI | Inspector「选取窗口」：点屏幕上的窗口，或从已打开窗口列表选择；**不要求用户手填进程名** |
| 说明 | 轮询直到窗口出现；超时 → `ok=false`，不中断整条流程 |

### 3.15 激活窗口 `window_activate`

| 项 | 内容 |
|---|---|
| 参数 / UI | 同 `window_wait`（选取窗口） |
| 输出 | `ok`, `title`, `pid`, `error` |
| 说明 | 还原最小化并前置；找不到 → `ok=false` |

### 3.16 关闭窗口 `window_close`

| 项 | 内容 |
|---|---|
| 参数 | 选取窗口 + `force`（默认正常关闭；可选强杀进程） |
| 输出 | `ok`, `title`, `pid`, `error` |

## 模块四：变量系统设计

- 每个有输出的Block，其输出自动以 `{node_id}.{output_name}` 的形式存入执行上下文（context），供后续任意节点在参数中通过 `{{node_id.output_name}}` 语法引用
- 前端画布上，节点之间除了"流程连线"（黑色箭头，表示执行顺序），还应支持"数据连线"（虚线，表示某节点的输出被拖到另一节点的参数上），点击数据连线可以看到具体引用了哪个字段
- 全局变量（用户手动定义，非节点输出）：提供一个"变量管理面板"，用户可预先定义变量初始值（如一个字符串"目标文字"），流程执行中被节点更新，也可被其他节点引用，命名建议加前缀区分，如 `$myVar` vs 节点输出 `{{node_id.output}}`

## 模块五：录制类功能

### 5.1 点击流程录制 `Recorder`

| 交互流程 | 说明 |
|---|---|
| 1. 用户点击"开始录制" | 前端调用 `api.start_recording()`，后端启动 `pynput` 全局鼠标/键盘监听 |
| 2. 用户在屏幕任意位置正常操作 | 后端记录每次点击的坐标、按钮类型、时间戳；键盘输入记录按键或文本 |
| 3. 用户点击"停止录制"（建议用固定快捷键触发，如 `Ctrl+Shift+F10`，避免录制窗口本身被记录进去） | 后端将记录的操作序列转换为对应的 `click`/`key_press`/`delay` 节点序列，`delay` 节点根据操作间的真实时间间隔自动生成 |
| 4. 生成的节点序列自动追加到当前画布 | 用户可在画布上进一步编辑（如把某个delay改成条件等待，或把固定坐标改成相对坐标） |

**异常处理**：
- 设置中「默认坐标基准」为 `window_client` 时，录制/取点会附带 `window_target`，并直接写出 `coordinate_mode: window_client`（捕获不到窗口时回退 `screen_abs`）
- 仍可选 `screen_abs` / `virtual_norm`；窗口移动后建议用 `window_client` 回放
- 提供"最小时间间隔合并"选项，避免手抖产生的密集重复点击都被记录成大量冗余节点

## 模块七：可视化编排界面

### 7.1 积木面板 `BlockPanel`

- 根据Block注册表自动渲染分类列表（动作类/识别类/控制类/系统类/自定义/触发类……）
- 支持搜索过滤
- 拖拽面板中的积木到画布，自动创建对应类型的默认参数节点

### 7.2 画布编辑器 `FlowCanvas`（基于React Flow）

- 节点：显示Block图标+名称+关键参数摘要（如click节点直接显示坐标）
- 连线：
  - 实线箭头 = 执行顺序（`next`/`then`/`else`/`body`）
  - 虚线 = 变量引用关系（可选显示/隐藏，避免复杂流程视觉混乱）
- 条件节点在画布上应有明显的"是/否"两个出口标签
- 循环节点应有视觉上的"循环体范围框"（可折叠/展开循环体内部节点）
- 支持节点多选、复制粘贴、框选删除
- 支持画布缩放、小地图导航（React Flow自带的Controls/MiniMap组件）

### 7.3 节点参数表单 `NodeParamForm`

- 点击画布节点后，右侧面板根据该节点类型的Schema自动生成对应表单控件（文本框/数字输入/颜色选择器/区域框选按钮/下拉选择）
- 涉及"坐标"/"区域"类参数，表单里提供"点击选取"按钮，点击后隐藏窗口、鼠标变十字光标，用户在屏幕上点选后自动回填

### 7.4 JSON代码编辑器 `CodeEditor`（基于Monaco）

- 语法高亮、自动格式化
- 实时Schema校验，错误行标红并悬浮提示具体错误信息（如"node_003缺少必填字段region"）
- 提供"格式化"、"校验"、"应用到画布"按钮（也可配置为输入防抖自动应用）

### 7.5 执行日志面板 `ExecutionLog`

- 实时滚动显示：当前执行节点ID+类型、输入参数、输出结果、耗时、报错堆栈（如有）
- 执行中的节点在画布上应高亮显示（如黄色边框），执行完成变绿色，出错变红色，便于用户直观追踪进度
- 支持日志导出为文本文件，便于问题排查

### 7.6 运行控制条

- 按钮：运行 / 暂停 / 停止 / 单步执行
- 单步执行模式下，每点击一次"下一步"，执行引擎只跑一个节点然后暂停，便于调试

## 模块八：项目管理

### 8.1 保存/加载

- 流程数据（FlowModel）序列化为 `.flow.json` 文件保存在用户指定目录
- 打开时校验文件Schema版本，如遇旧版本格式提供自动迁移逻辑（预留versionfield）

### 8.2 导入/导出

- 支持导出为 `.flow.json`，或含模板图的便携包 `.flow.zip`（内含 `flow.json` + `assets/`）
- 流程含找图/锚点模板等本地图片时，导出默认建议 zip；导入 zip 时图片写入本机 `templates/` 并重写节点路径
- 纯 JSON 导入导出仍可用（不带图片资源）

## 功能优先级总览（对应需求文档MVP划分）

| 优先级 | 功能 |
|---|---|
| P0（MVP必须） | click/key_press/delay、color_detect/if_color_match、if_condition/switch、loop_n/loop_while、画布拖拽、JSON编辑器双向同步、参数表单、执行引擎、运行控制、执行日志、保存/加载 |
| P1（第二期） | ocr_recognize/if_text_contains、find_image、schedule_trigger、call_subflow |
| P1（系统 IO） | http_request、clipboard、file_io、run_command、notify |
| P1（窗口） | window_wait、window_activate、window_close；录制/拖拽 `window_client` 闭环 |
| P1（F-25） | python_script、user_blocks 自定义积木目录 |
| P1（打包/调度） | 流程 `.flow.zip` 导入导出；定时任务落盘与启动恢复；断点变量监视 |
| P2（第三期/远期） | 多显示器混 DPI 产品化 |
